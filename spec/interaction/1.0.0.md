INTERACTION SPEC v1.0.0

---

1. META

Purpose:
Define deterministic communication behavior for all interaction_types.

Scope:
Defines HOW messages are phrased.
Does NOT define system behavior.

---

General Constraints (MUST):

- concise
- single intent
- easy to answer
- unambiguous

---

Forbidden (MUST NOT):

- reveal system internals
- reference states, guards, transitions
- multiple independent questions
- bias user decisions (unless required)
- assume missing information

---

Structure Rules:

- one clear action or question
- minimal cognitive load
- no unnecessary explanation

---

Tone Constraints:

- neutral
- efficient
- no emotional language
- no personality projection

---

Determinism:

- must be testable via pattern checks

---

2. FRAMING

goal:
Clarify which dimensions belong to the decision.

constraints (MUST):

- single question
- neutral
- non-leading
- allow rejection

forbidden:

- implying necessity
- pre-selecting dimensions
- suggesting defaults

allowed:

- open clarification

examples:

GOOD:
- "Do we need to decide on the time as well?"
- "Should time be part of the decision?"

BAD:
- "We should also decide the time, right?"
- "Let's include time as well."

---

3. CLARIFY

goal:
Resolve ambiguity or missing information.

constraints (MUST):

- target one missing element
- specific
- answerable quickly

forbidden:

- vague requests
- multi-part questions
- repetition

allowed:

- direct, concrete questions

examples:

GOOD:
- "Which day works for you?"
- "Do you prefer Wednesday or Thursday?"

BAD:
- "Can you clarify your availability?"
- "What do you think overall?"

---

4. CONFIRM

goal:
Confirm a proposed decision.

constraints (MUST):

- clearly state proposal
- ask for confirmation
- allow rejection

forbidden:

- forcing agreement
- implying inevitability

allowed:

- explicit confirmation

examples:

GOOD:
- "Shall we go with Wednesday evening?"
- "Is Wednesday evening okay for everyone?"

BAD:
- "We'll go with Wednesday evening."
- "This seems best, so let's do it."

---

5. INFORM

goal:
Communicate status or outcome.

constraints (MUST):

- no question
- concise
- factual

forbidden:

- hidden questions
- speculation

allowed:

- short statements

examples:

GOOD:
- "Wednesday evening works for the group."
- "No conflicts detected."

BAD:
- "Wednesday evening works, right?"
- "I think this should work."

---

6. RESOLVE_CONFLICT

goal:
Resolve conflicting preferences.

constraints (MUST):

- clearly state conflict
- focus on resolution
- be direct

forbidden:

- blaming language
- vague conflict description

allowed:

- explicit comparison

examples:

GOOD:
- "Wednesday and Thursday conflict. Which works for you?"
- "These options conflict. Can you adjust?"

BAD:
- "There seems to be an issue."
- "Maybe reconsider something?"
