1. System Goal

Guide a group from an open question to a clear decision as efficiently as possible.

---

2. Core System Model

System = EFSM (Extended Finite State Machine)

Statechart is:
- first-class artifact
- executable
- single source of truth

---

3. Architecture

User Input  
→ LLM Interpretation  
→ Structured Signals  
→ Events  
→ State Machine  
→ Actions  
→ Targeting Layer  
→ Interaction Type Selection

4. Role of the LLM

LLM:
- interprets input
- produces signals
- generates language
- refines interaction phrasing

LLM does NOT:
- execute transitions
- override guards
- select targets
- select interaction_type

Rule:
LLM suggests, system decides

==================================================
36.10 INTERACTION TYPE SELECTION (NEW)
==================================================

interaction_type = f(state, context, targeting_reason)

---

36.10.1 Baseline Mapping

IF state == clarifying:
    interaction_type = clarify

IF state == collecting:
    IF missing_preferences:
        interaction_type = preference
    ELSE IF missing_constraints:
        interaction_type = constraint
    ELSE:
        interaction_type = clarify

IF state == resolving:
    interaction_type = resolve_conflict

IF state == deciding:
    IF confirmation_required:
        interaction_type = confirm
    ELSE:
        interaction_type = inform

---

36.10.2 Targeting Reason Refinement

- missing_constraint → constraint  
- missing_preference → preference  
- conflict_participant → resolve_conflict  
- uncertainty_only → clarify  

---

36.10.3 Fallback

interaction_type = inform

---

36.10.4 Constraints

Must be:
- deterministic
- testable
- state-consistent

---

36.10.5 LLM Constraint

LLM must NOT override interaction_type
