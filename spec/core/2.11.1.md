40.12 FRAMING INTERACTION CONSTRAINTS (NEW)
==================================================

40.12.1 Minimal Interaction Principle

The system MUST minimize framing interactions.

Framing MUST only occur if:

- trigger conditions are met
AND
- framing provides material decision value

Material decision value exists if:

- framing may influence expected_dimensions
AND
- expected_dimensions would affect completeness routing

The system MUST avoid unnecessary clarification.

---

40.12.2 Non-Repetition Constraint

Framing MUST NOT be repeated within the same decision context.

Exception (future extension only):

- new materially relevant dimensions emerge

---

40.12.3 Non-Blocking Continuation Principle

After framing is triggered:

- the system MUST continue decision progression
- the system MUST NOT wait for initiator response

If no response is received:

→ expected_dimensions remains None  
→ system proceeds without dimension-based completeness

---

40.12.4 Neutrality Constraint

Framing interactions MUST:

- remain neutral
- avoid suggesting required structure
- avoid implying that dimensions are mandatory

The system MUST NOT:

- bias the initiator toward accepting proposed_dimensions
- imply that proposed dimensions are required

---

40.12.5 Non-Leading Formulation Principle

Framing questions MUST:

- not embed assumptions
- not pre-select dimensions
- allow rejection or modification

The system MUST NOT:

- present proposed_dimensions as default
- imply necessity (e.g. "we should also decide X")

---

40.12.6 Brevity Constraint

Framing interactions SHOULD be:

- concise
- low cognitive load
- single-purpose

The system SHOULD avoid multi-part or compound framing questions.

---

40.12.7 Separation Principle

These constraints apply to:

- Targeting Layer behavior
- Interaction Type Selection

They MUST NOT:

- affect EFSM transitions
- affect guards
- affect decision validity
