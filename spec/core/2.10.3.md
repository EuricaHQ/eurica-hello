25. ACTION LAYER

The Action Layer is responsible for:

==================================================
25.7 SIGNAL → CONTEXT MAPPING (NEW)
==================================================

The system must transform LLM-derived signals into structured context
before guard evaluation.

Signals must NOT be used directly by guards.

---

25.7.1 Principle

Signal Layer:
→ detects semantic conditions

Action Layer:
→ materializes signals into context

Guards:
→ operate exclusively on context

---

25.7.2 Conflict Mapping Rule

IF signal.conflict == true:

→ create a conflict entry:

conflict = {
  participants: [actor],
  type: optional,
  source: "llm_signal",
  timestamp: now,
  status: "open"
}

append to ctx.conflicts

---

25.7.3 Minimal Required Structure

ctx.conflicts must be a list of:

conflict = {
  participants: List[participant_id],
  source: string,
  timestamp: datetime,
  status: "open" | "resolved"
}

Optional:
- type (preference / constraint / other)

---

25.7.4 Optional Enrichment

LLM may provide:

- additional participants
- conflict type

System must function without enrichment.

---

25.7.5 Deduplication Rule

Before adding a conflict:

IF a similar conflict exists with:
- overlapping participants
- recent timestamp
- status == "open"

→ do NOT add duplicate

---

25.7.6 Conflict Lifecycle

Creation:
→ on signal.conflict == true

Persistence:
→ remains in ctx.conflicts while unresolved

Resolution:
→ conflict must be:
   - removed OR
   - marked as "resolved"

when:
- conflict no longer affects solution
- participants align
- constraints are reconciled

---

25.7.7 Guard Dependency

has_conflict must evaluate:

len(ctx.conflicts where status == "open") > 0

Guards must NOT use signal.conflict directly.

---

25.7.8 LLM Constraint

LLM:
- may detect conflict
- may enrich conflict

LLM must NOT:
- modify ctx.conflicts
- create or delete context entries

---

25.7.9 Generalization Principle

This mapping pattern applies to all signals:

signal → structured context → guard evaluation

Conflict mapping is the primary required implementation.

Other signal mappings may be introduced in future versions.
